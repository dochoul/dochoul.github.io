대우건설 푸르지오 통합 사이트 리뉴얼 프로젝트에 프론트엔드로 참여했다. PC 사이트(<a href="https://www.prugio.com" target="_blank" rel="noopener">www.prugio.com</a>)와 모바일 사이트(<a href="http://m.prugio.com" target="_blank" rel="noopener">m.prugio.com</a>)를 함께 다시 만들면서, 이번에는 디자인이나 반응 속도가 아니라 웹 접근성을 최우선 목표로 잡았다.

## 프로젝트 개요

- **기간:** 2013.02 ~ 2013.04
- **역할:** 프론트엔드 개발
- **소속:** 더위버크리에이티브
- **범위:** 한국형 웹 콘텐츠 접근성 지침(KWCAG) 2.0 준수 개발, Masonry 기반 동적 레이아웃, Image Sprite·JSON 데이터 기반 웹진 운영 구조

## 왜 접근성이었나

아파트 브랜드 홈페이지 중에 웹 접근성 인증마크를 받은 곳이 아직 없었다. 스크린리더 사용자나 키보드만으로 사이트를 탐색하는 사용자도 콘텐츠를 동등하게 이용할 수 있어야 한다는 요구가 이번 리뉴얼의 출발점이었다. 그래서 디자인을 다 확정한 뒤에 접근성을 끼워 맞추는 방식이 아니라, 개발 기준으로 삼은 <a href="https://www.websoul.co.kr/accessibility/WA_guide21.asp" target="_blank" rel="noopener">한국형 웹 콘텐츠 접근성 지침(KWCAG) 2.0</a>을 마크업 단계부터 원칙으로 잡았다.

- 이미지 콘텐츠에는 스크린리더가 읽을 수 있도록 대체 텍스트를 빠짐없이 붙였다.
- 마우스 없이 키보드만으로 전체 네비게이션을 이동할 수 있도록 포커스 순서와 탭 이동을 정리했다.
- 저시력자와 고령자를 고려해 텍스트와 배경의 명도대비를 개선했다.

## 24개 검사항목과의 싸움

말이 좋아 "지침 준수"지, 실제로는 인식의 용이성·운용의 용이성·이해의 용이성·견고성이라는 4대 원칙 아래 13개 지침, 그 아래 20여 개 세부 검사항목을 사이트 구석구석에 전부 적용하는 일이었다. 메인 페이지 하나만 잘 만들어서 될 일이 아니라, 대우건설 통합 사이트 안의 모든 서브 페이지와 웹진 이슈 페이지까지 예외 없이 같은 기준을 통과해야 했다.

특히 힘들었던 건 항목 하나하나는 별거 아닌데, 그걸 페이지 수만큼 곱하면 작업량이 기하급수적으로 늘어난다는 점이었다. 이미지 하나에 대체 텍스트를 붙이는 건 5분이면 끝나지만, 콘텐츠가 수백 개인 웹진 아카이브 전체에 붙이려면 얘기가 달라진다. 키보드 포커스 순서도 마찬가지였다. 새로 만든 페이지는 처음부터 순서를 신경 써서 짜면 되지만, 기존에 있던 화면들은 마크업 구조 자체를 다시 들여다보며 탭 순서를 하나씩 바로잡아야 했다.

## Masonry 레이아웃과 접근성을 함께 가져가기

콘텐츠 카드 크기가 제각각인 페이지에는 Masonry 기반의 동적 레이아웃을 적용했다. 문제는 이런 동적 배치가 자칫 마크업 순서와 화면에 보이는 순서를 어긋나게 만들어, 스크린리더로 읽었을 때 순서가 뒤죽박죽될 수 있다는 점이었다. 시각적으로는 자유롭게 배치하되, DOM 순서와 포커스 이동 순서는 실제 콘텐츠 흐름과 일치하도록 맞추는 데 신경을 썼다.

## 웹진 운영 편의를 위한 구조

매달 새 이슈가 올라오는 웹진 특성상, 담당자가 매번 이미지를 하나하나 새로 잘라 넣는 방식은 운영 부담이 컸다. 반복되는 UI 요소는 Image Sprite로 묶고, 이슈별 콘텐츠 데이터는 JSON으로 분리해 관리하도록 만들었다. 덕분에 매달 업데이트할 때 마크업을 새로 손대지 않고 JSON 데이터만 갱신하면 되는 구조가 됐다.

## 심사를 통과하기까지

개발이 끝났다고 인증이 바로 나오는 게 아니었다. 한국웹접근성인증위원회(KWAC)의 심사는 체크리스트에 표시만 하고 끝나는 형식적인 절차가 아니라, 실제로 스크린리더를 켜고 페이지를 하나씩 훑으며 검증하는 과정이었다. 서면으로 자료를 정리해 제출하고, 심사 과정에서 미흡하다고 지적된 부분을 다시 고쳐 재제출하는 일이 반복됐다. 개발 단계에서 이미 충분히 챙겼다고 생각한 부분에서 지적이 나올 때마다, 어디까지를 "충분한 접근성"으로 봐야 하는지 기준을 다시 잡아야 했다.

마감을 코앞에 두고 이 과정을 반복하다 보니, 이번 프로젝트 중에서 체력적으로나 정신적으로나 가장 힘들었던 구간이었다. 그만큼 최종 인증마크를 받았을 때의 체감도 남달랐다.

## 결과

인증마크는 4월 30일에 먼저 확정됐고, 5월 8일 보도 자료를 통해 대외적으로 알려졌다. 숙명여대 웹발전연구소와 지속가능과학회, ITNEWS가 공동 구성한 한국웹접근성인증위원회(KWAC)로부터 받은 인증으로, 아파트 브랜드 홈페이지로는 처음이었다.
