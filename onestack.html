What is Onestack

대화 한 마디로 배포가
끝나는 플랫폼

Onestack은 AI 에이전트가 로컬에서 만든 프로젝트를 즉시 라이브 서비스로 만드는 배포 플랫폼입니다. 콘솔에서 직접 설정할 수도 있지만, 에이전트에게 한 마디만 해도 분석·구성·배포·검증을 자동으로 마칩니다.

모든 배포는 <app-slug>.onestack.run으로 자동 라우팅되고, 배포 후 상태와 응답까지 검증해 결과를 알려줍니다.

Capabilities

한 마디로 끝나는 배포 작업

에이전트가 Onestack 위에서 자동으로 처리하는 기능들

프로젝트 자동 분석

Git 상태, 프레임워크, Dockerfile, Compose, 환경 파일을 탐지해 Onestack 리소스 타입을 자동 추천합니다.

빌드 타입 자동 매핑

프로젝트 구조에 맞춰 static, nixpacks, dockerfile, compose 중 가장 안전한 경로를 자동 선택합니다.

리소스 자동 생성

Project · Environment · Application · Compose · Database를 검색하고 없으면 생성, 있으면 재사용합니다.

자연어 명령

"배포해줘" · "재배포해줘" · "도메인 바꿔줘" 같은 자연어 요청으로 모든 작업이 실행됩니다.

Managed Database

Postgres, MySQL 등을 관리형 서비스로 분리해 앱과 내부 서비스 호스트네임으로 연결합니다. 직접 컨테이너 관리가 필요 없습니다.

로컬 Docker 불필요

ZIP Drop 업로드 경로를 지원해 로컬에 Docker가 없어도 배포가 가능합니다. 빌드는 서버에서 수행됩니다.

배포 후 자동 검증

배포 큐, 리소스 상태, 공개 URL 응답을 자동 확인합니다. 직후 502는 짧은 retry 윈도우 안에서 재시도합니다.

도메인 자동 라우팅

<app-slug>.onestack.run으로 자동 등록되며, 커스텀 도메인도 에이전트 요청으로 추가할 수 있습니다.

How it works

3단계로 끝나는 시작

설치부터 배포까지, 복잡한 설정 없이 바로 시작합니다

1

스킬 설치

AI 에이전트에 Onestack 스킬을 설치합니다. 한 번 설치하면 모든 프로젝트에서 사용 가능합니다.

2

API 키 발급

Onestack 콘솔에서 API 키를 발급받아 환경 변수에 설정합니다. 에이전트가 콘솔과 통신하는 통로입니다.

3

한 줄 요청

프로젝트 폴더에서 "Onestack에 배포해줘"라고 요청하면 분석부터 배포·검증까지 자동 실행됩니다.

Deployment Workflow

에이전트가 실행하는 8단계

배포 요청 한 마디가 트리거하는 전체 워크플로우

1

프로젝트 분석

Git 상태, 프레임워크, Dockerfile, Compose, 환경 파일을 식별하고 최적 빌드 타입을 결정합니다.

2

Git 상태 점검

원격 저장소·브랜치·푸시 상태를 확인합니다. 미푸시 변경은 git-backed 배포에 반영되지 않으므로 사전에 안내합니다.

3

Onestack 인증

API 키와 세션을 검증합니다. 실패하면 작업을 멈추고 콘솔에서 키를 발급하는 절차를 안내합니다.

4

Project / Environment 확보

기존 Onestack 프로젝트를 검색하고, 없으면 새로 만든 뒤 prod 환경을 선택합니다.

5

Application / Compose 생성

빌드 타입에 맞는 리소스를 생성하거나 기존 리소스를 재사용합니다.

6

리소스 구성

소스, 빌드 옵션, 환경 변수, 도메인 *.onestack.run, 포트를 등록합니다.

7

배포 실행

Application은 ZIP Drop으로, Compose는 Compose Deploy로 트리거합니다. 배포 큐가 처리될 때까지 대기합니다.

8

상태 검증

큐 상태, 리소스 상태, 공개 URL 응답을 확인합니다. 직후 일시적 502는 짧은 retry 윈도우 안에서 재확인 후 결과를 보고합니다.

Validated Paths

검증된 배포 경로

실제 end-to-end 테스트를 통과한 경로만 사용합니다

Validated

Application · Drop · Static

사전 빌드된 정적 자산. ZIP 업로드 후 공개 URL 200 OK 검증 완료.

Validated

Application · Drop · Nixpacks + Managed Postgres

서버 앱 + Onestack 관리형 Postgres. DB-backed HTTP 응답까지 end-to-end 검증.

Validated

Compose · Raw

이미지 기반 멀티 서비스 Compose 스택. 공개 URL 200 OK 검증.

Safety

안전 원칙

Onestack 에이전트가 지키는 보안 원칙

시크릿 비노출

API 토큰, DB 비밀번호, .env 값, 시크릿이 포함된 명령 전체를 응답에 노출하지 않습니다.

삭제 금지 (승인 전)

Application, Database, Domain, Volume, Deployment는 사용자가 명시적으로 승인하기 전까지 삭제하지 않습니다.

Git 상태 점검

Onestack은 원격 소스 기준으로 배포하므로 배포 전 미커밋·미푸시 변경을 사용자에게 알립니다.

구조화된 결과 파싱

가능한 경우 JSON 응답을 사용해 상태와 결과를 안정적으로 처리합니다.

인증 위조 금지

API 키가 없으면 임의 값을 만들지 않고, 사용자에게 키 제공 또는 콘솔 생성을 안내합니다.

실패 시 중단

인증·리소스 생성·배포 중 실패하면 즉시 멈추고 원인과 해결 방법을 보고합니다.

자주 묻는 질문

Onestack은 로컬에서 만든 코드 프로젝트를 라이브 서비스로 만들어주는 배포 플랫폼입니다. AI 에이전트와 결합해 "한 마디로 배포"가 가능하도록 설계되었습니다. 콘솔은 https://console.onestack.run에서 사용할 수 있습니다.
아니요. 설치 스크립트는 curl 또는 irm으로 GitHub에서 직접 다운로드하므로 Git이 필요 없습니다.
Claude Code, Cursor, Codex CLI, Gemini CLI, OpenCode 등 ~/.claude/skills 또는 .agents/skills를 인식하는 에이전트라면 모두 사용할 수 있습니다. 설치 후 에이전트에게 "Onestack으로 배포해줘"라고 요청하면 스킬이 자동으로 활성화됩니다.
Onestack 콘솔(https://console.onestack.run)의 User Settings → API/CLI 메뉴에서 발급합니다. 발급된 키를 환경 변수 ONESTACK_API_KEY에 설정하면 됩니다.
네. Onestack은 ZIP Drop 업로드 경로를 지원하므로 빌드가 서버에서 일어납니다. 로컬에 Docker가 설치돼 있지 않아도 배포에 문제가 없습니다.
이미지 기반의 self-contained Compose라면 Raw Compose 경로로 그대로 배포할 수 있습니다. 로컬 빌드 컨텍스트가 필요한 Compose는 원격 소스 접근이 가능한 경우에만 안전하게 동작합니다.
Onestack은 Postgres, MySQL 등을 관리형 서비스(Managed DB)로 분리해 내부 서비스 호스트네임으로 연결합니다. docker-compose에 DB를 묶지 않는 것이 기본입니다.
기본 공개 도메인은 <app-slug>.onestack.run이며 Traefik이 자동 라우팅합니다. 별도 도메인을 사용하려면 콘솔이나 에이전트 요청으로 추가 등록하면 됩니다.
리소스가 완전히 기동하기 전 짧은 시간 동안 502가 발생할 수 있습니다. 에이전트는 짧은 retry 윈도우 안에서 자동 재확인하며, 지속되면 실패로 보고합니다.
