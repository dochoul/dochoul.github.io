### 비서를 한 명 들인 이야기

우리 팀 채널엔 매일 아침 뉴스가 도착한다. 보내는 건 사람이 아니라 **가비아에이전트**다.  
매일 오전 8시, Hada News에서 최신 기사를 가져와 채널에 올린다. 자리에 앉으면 뉴스가 이미 와 있다.

### 가비아에이전트의 동선

동작 순서는 간단하다.

```
┌──────────────────────┐     ┌─────────────────┐     ┌──────────────────────┐
│  Hada News RSS Feed  │ ──> │   Cron Job      │ ──> │  Mattermost Webhook  │
│  (매일 오전 8시 출근)    │     │ (가비아에이전트)    │     │  (#소프트솔루션유닛)     │
└──────────────────────┘     └─────────────────┘     └──────────────────────┘
```

1. 오전 8시(KST), Cron Job이 실행된다.
2. `https://news.hada.io/rss/news`에서 RSS 피드를 받아 온다.
3. 뉴스 제목과 링크를 마크다운 리스트로 정리한다.
4. Mattermost 웹훅으로 POST 요청을 보낸다.
5. 채널에 메시지가 올라온다.


![채널에 올라온 뉴스 메시지](/assets/geek-news-bot/geek-news.png){: style="border: 1px solid #e1e5e9; border-radius: 6px;"}

### 가비아에이전트의 신상 명세서

설정값은 이렇다.

| 항목 | 내용 |
| --- | --- |
| Job ID | `a16817232a1a` |
| 이름 | Hada News to Mattermost |
| 근무 시간 | `0 8 * * *` (매일 오전 8시) |
| RSS 출처 | news.hada.io/rss/news |
| 웹훅 URL | mattermost.gabia.com/hooks/`k4qi…g3fh` |
| 모델 | qwen3 |
| 장비 | web, terminal |
| 등록일 | 2026-05-29 |

> 웹훅 URL은 주소만 알면 누구나 채널에 글을 쓸 수 있기 때문에, 위에는 일부러 가운데를 가렸다. 외부에 공개하는 글이라면 통째로 노출하지 않도록 주의하자.

![가비아 에이전트에 등록된 cron job 화면](/assets/geek-news-bot/gabia-agent.png){: style="border: 1px solid #e1e5e9; border-radius: 6px;"}

### 가비아에이전트에게 내린 지시문

에이전트라고 알아서 다 하는 건 아니다. 일을 시키려면 또박또박 일러줘야 한다.

> 1. RSS 피드를 가져와라.
> 2. 최신 뉴스의 제목과 링크를 추출해라.
> 3. `[제목](링크)` 형태의 리스트로 보기 좋게 정리해라.
> 4. 정리한 내용을 Mattermost 웹훅으로 보내라.
> 5. JSON 페이로드의 `text` 필드에 담아 POST로 전송해라.
> 6. 새 뉴스가 없으면 메시지를 보내지 마라.

마지막 6번이 중요하다. 새 뉴스가 없을 때 빈 메시지를 보내지 않도록 하는 조건이다.

### 가비아에이전트를 한 명 더 들이고 싶다면

다른 채널에도 추가하고 싶을 때의 순서다.

**1단계 — 웹훅 만들기**
Mattermost 관리자 설정 → Integration → **Incoming Webhooks**에서 새 웹훅을 만든다. 입력칸은 이렇게 채우면 된다.

| 항목 | 무엇을 적나 | 예시 |
| --- | --- | --- |
| 제목 | 웹훅 설정을 알아볼 이름 (최대 64자) | `gabia-agent-geek-news-bot` |
| 설명 | 이 웹훅이 무슨 용도인지 메모 | `매일 아침 GeekNews를 채널로 전송` |
| 채널 | 뉴스를 받을 기본 채널 | `소프트솔루션개발유닛` |
| 이 채널로 고정 | 체크하면 다른 채널엔 못 올리고 선택한 채널에만 게시된다. 켜 두는 걸 권장 | ✅ |
| 사용자 이름 | 메시지를 올릴 때 표시될 이름 (선택, 비우면 생성자 이름 사용) | 비워 둠 |
| 프로필 사진 | 표시될 프로필 이미지 URL, `.png`/`.jpg`, 최소 128×128 (선택) | 비워 둠 |

![Mattermost Incoming Webhooks 설정 화면](/assets/geek-news-bot/mattermost-webhook.png){: style="border: 1px solid #e1e5e9; border-radius: 6px;"}

다 채웠으면 **업데이트(저장)**를 누른다. `https://mattermost.gabia.com/hooks/...` 형태의 웹훅 URL이 발급된다. 복사해 두자.

> ⚠️ "비공개 채널"에 웹훅을 걸려면, 설정하는 사람이 먼저 그 채널의 멤버여야 한다. 그리고 이 URL은 외부에 노출하지 말 것 — 주소만 알면 누구나 그 채널에 글을 쏠 수 있다.

**2단계 — 에이전트에게 일 시키기**
에이전트에게 이렇게만 말하면 된다.

> 매일 오전 8시에 `https://news.hada.io/rss/news` RSS 피드를 가져와서
> 마크다운으로 정리하고, 아래 Mattermost 웹훅으로 보내줘:
> `https://mattermost.gabia.com/hooks/생성된웹훅URL`

**3단계 — 잘 들어왔는지 확인**
`cronjob action=list`로 job이 등록됐는지, 스케줄이 `0 8 * * *`인지, 웹훅 URL이 맞는지 확인한다.

### 뉴스가 안 올라올 때

아침에 뉴스가 안 보이면 순서대로 확인한다.

먼저 `cronjob action=list`로 상태를 본다. `last_status`가 `error`면 에러 메시지부터 읽는다. RSS 길이 막힌 건 아닌지 직접 가서 확인하려면:

```bash
curl -s https://news.hada.io/rss/news | head -20
```

피드는 멀쩡한데 전송이 안 됐다면 웹훅 URL이 유효한지(Mattermost 설정에서) 확인한다.

**잠깐 쉬게 하거나 다시 부를 때**는 말로 하면 된다. "Hada News cron job을 일시 정지해줘" 혹은 "재개해줘"라고 에이전트에게 부탁하면 그만이다. 출처나 시간, 포맷을 바꾸고 싶을 때도 마찬가지. 프롬프트를 직접 손봐야 하는 경우엔 `cronjob action=update`를 쓴다.

### 참고로 남기는 주소들

- Hada News: https://news.hada.io
- RSS 피드: https://news.hada.io/rss/news

코드를 작성하지도, 서버를 구성하지도 않았다. 에이전트에게 "매일 아침 이 RSS 가져와서 우리 채널에 보내줘"라고 말한 게 전부다.

매일 반복하는 귀찮은 일이 하나쯤 있다면, 그걸 에이전트에게 맡겨 보는 것부터 시작하면 된다.
