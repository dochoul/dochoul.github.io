## 생일 축하해요, AI가 — gabia-birthday 프로젝트 회고

> "이번 달 생일자 기프티콘 발송 준비해 주세요."

매달 반복되는 이 한 줄의 요청. 누군가는 엑셀을 열고, 전체 직원 명부에서 이번 달 생일자를 하나씩 눈으로 찾아가며 목록을 만들었을 것이다. 그리 복잡한 일은 아니지만, 그렇다고 신나는 일도 아닌 그런 업무.

이 작은 반복 업무에서 이 프로젝트는 시작됐다.

---

## AI 시대의 업무 재편, 그리고 작은 출발

요즘 개발 현장에서는 "에이전틱 코딩(Agentic Coding)"이라는 말이 자연스럽게 오간다. AI가 단순히 코드 자동완성을 돕는 수준을 넘어, 실제 업무 흐름 안에서 주도적으로 작동하는 방식이다. Claude, Cursor, Copilot 같은 도구들이 곁에 있으니, 이전에 사람이 직접 처리하던 반복적인 공정들을 AI에게 넘기는 게 점점 자연스러워졌다.

우리 팀에도 그런 미션이 생겼다. 기존 회사 업무 중 AI가 더 잘 할 수 있는 것들을 찾아내고, 실제로 동작하는 형태로 만들어 현업에 전달하는 것. 이건 단순히 "AI 써봤어요"가 아니라, 실제로 누군가의 업무 시간을 돌려주는 일이었다.

그 첫 번째 대상으로 고른 게 바로 **임직원 생일 관리** 업무였다.

---

## 문제는 단순했다, 그래서 더 좋았다

인사팀에서는 매달 이런 일을 한다.

1. 전체 임직원 엑셀 명부를 열기
2. 이번 달 생일자만 필터링하기
3. 고용형태 확인 (정규직만 기프티콘 지급, 수습/인턴/휴직은 제외)
4. 발송용 엑셀 파일 정리해서 담당자에게 전달

딱 봐도 자동화하기 좋은 구조다. 규칙이 명확하고, 입력값(엑셀)과 출력값(필터링된 명단)이 정해져 있으니까.

그런데 여기서 변수가 하나 있었다. 인사팀 원본 엑셀에는 **주민등록번호**가 포함돼 있다는 것. 웹 서비스로 만들어서 어딘가에 올려두면 개인정보 보호 측면에서 부담이 생긴다. 그래서 방향을 바꿨다.

---

## 1단계: 웹으로 만들었다가, 다시 데스크톱 앱으로

처음 접근은 **웹 서비스**였다. 브라우저에서 바로 접근할 수 있으니 담당자 입장에서 가장 쉬운 방법이기도 했다.

그런데 금방 문제가 보였다. 인사팀 엑셀에는 **주민등록번호**가 포함돼 있다. 웹 서비스로 올리는 순간 개인정보가 서버를 오가게 되고, 그건 부담이었다.

그래서 방향을 틀었다. **Electron 기반 데스크톱 앱**으로 다시 만든 것이다. 서버에 올리지 않고 담당자 컴퓨터에서만 돌아가니 개인정보가 외부로 나갈 일이 없다. 게다가 앱 안에서 주민등록번호는 **업로드 즉시 삭제**하고 생일(YYYY-MM-DD)만 추출해서 저장하도록 설계했다.

기술 스택은 이렇게 구성했다.

| 영역       | 기술                                   |
| ---------- | -------------------------------------- |
| 데스크톱   | Electron 35, electron-builder          |
| 프론트엔드 | React 18, TypeScript, Vite, Mantine v8 |
| 백엔드     | Express 4, TypeScript                  |
| 엑셀 처리  | xlsx, ExcelJS                          |

구조는 모노레포로 `client/`, `server/`, `electron/` 세 파트로 나뉜다. 로컬에서 프론트와 백이 함께 돌아가는 형태다.

주요 기능은 이렇다.

- **월별 생일자 목록** 조회 (생일 오름차순 정렬)
- **엑셀 내보내기** — 팀/유닛 구분이 포함된 `.xlsx` 파일 다운로드
- **직원 명부 업로드** — 드래그&드롭 또는 클릭으로 업로드
- **업로드 내역** — 파일명, 일시, 고용형태별 인원 수 기록
- **월별 통계 차트** — 고용형태별 생일자 분포 시각화
- **기프티콘 지급 대상 필터링** — 정규직(재직중)만 활성화

macOS와 Windows 둘 다 빌드해서 DMG와 exe 파일을 각각 담당자에게 전달했다. 처음 실행할 때 "확인되지 않은 개발자" 경고가 뜨는 건 어쩔 수 없지만, 그것도 README에 해결 방법까지 써뒀다.

## 2단계: Claude 스킬로

데스크톱 앱을 만들고 나서 한 가지 생각이 들었다.

> "담당자가 매달 앱을 열고, 월을 선택하고, 버튼 누르는 게 여전히 조금은 번거롭지 않을까?"

그래서 한 발 더 나아갔다. **Claude 스킬(Skill)** 로 만든 것이다.

Claude에 연결된 스킬로 만들어두면, 담당자가 그냥 대화하듯이 물어보면 된다.

> "이번 달 생일자 목록 뽑아줘"
> "4월 기프티콘 대상자 엑셀로 내보내줘"

앱을 설치하거나 실행할 필요도 없다. Claude와 대화하는 것만으로 업무가 처리된다. 기존 도구(Claude.ai)를 이미 쓰고 있는 사람이라면 진입 장벽이 거의 없다.

스킬에는 다음 기능들을 담았다.

- 월별 생일자 목록 조회
- 기프티콘 지급 대상자 엑셀 내보내기
- 고용형태별 통계
- 직원 명단 조회

---

## 3단계: Claude for Excel로

그런데 또 한 가지 현실적인 문제가 있었다. 개인 생일 정보가 담긴 엑셀 파일은 결국 인사팀 로컬에 있고, 담당자들은 이미 엑셀 환경에 익숙하다는 것.

그래서 세 번째 접근으로 **Claude for Excel**을 도입했다.

엑셀 안에서 Claude를 직접 활용하는 방식이다. 별도 앱이나 서비스로 이동하지 않아도, 이미 열려 있는 엑셀 파일 위에서 바로 AI에게 물어보고 필터링하고 정리할 수 있다. 익숙한 환경 그대로, 거기에 AI가 얹히는 구조.

---

## 세 가지 버전, 하나의 목표

정리해보면 이 프로젝트는 같은 문제를 세 가지 방법으로 풀었다.

| 버전 | 형태                      | 특징                                   |
| ---- | ------------------------- | -------------------------------------- |
| v1   | 웹 → Electron 데스크톱 앱 | 개인정보 이슈로 웹에서 전환, 로컬 처리 |
| v2   | Claude 스킬               | 대화형, 설치 불필요, 자연어 인터페이스 |
| v3   | Claude for Excel          | 엑셀 환경 그대로, 익숙한 도구 위에 AI  |

세 버전 모두 목표는 같다. **매달 반복되는 단순 업무에서 담당자를 해방시키는 것.**

## 회고: 작은 자동화가 주는 것

이 프로젝트를 하면서 가장 인상 깊었던 건, "작은 문제"를 제대로 파고들었을 때 생각보다 깊은 설계가 필요했다는 점이다.

단순히 엑셀 읽어서 필터링하는 건 금방이다. 하지만 여기엔 이런 것들을 고려해야 했다.

- 주민등록번호를 어떻게 안전하게 처리할 것인가
- 담당자가 비개발자라면 어떤 인터페이스가 편할까
- 데스크톱 앱이냐, AI 스킬이냐, 엑셀 안이냐 — 어떤 형태가 실제로 쓰일까
- 인사팀 엑셀 포맷이 바뀌면 어떻게 대응할까

결국 좋은 자동화는 코드 품질보다 **사용자의 맥락을 얼마나 이해했느냐**에 달려 있다는 걸 다시 한 번 확인했다.

그리고 AI 에이전틱 코딩 덕분에 이 모든 것을 혼자서도 빠르게 구현할 수 있었다. 예전이라면 디자이너, 백엔드 개발자, QA 등 여러 사람이 손을 거쳐야 했을 것들을 AI와 둘이서 빠르게 치고 나갔다.

## 마치며

"AI로 뭔가를 만든다"는 게 거창한 일이어야 할 필요는 없다. 누군가의 매달 30분을 아껴주는 것, 엑셀에서 눈으로 찾던 생일자를 한 번의 대화로 뽑아주는 것 — 그걸로도 충분히 가치 있는 프로젝트다.

그 작은 가치들이 쌓이면, 회사 전체의 업무 방식이 조용히 달라진다.

앞으로도 그런 작은 자동화들을 하나씩 만들어 나갈 생각이다.
