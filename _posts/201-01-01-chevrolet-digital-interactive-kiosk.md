2011년 2월부터 6월까지, 다섯 달 동안 **쉐보레 디지털 인터랙티브 키오스크(DIK)** 프로젝트에 매달렸다. 46인치 대형 터치 스크린 위에서 쉐보레 브랜드를 직관적으로 전달하는 게 목표였고, 끝나고 나서는 2011 앤어워드(&Award) Digital Appliance UI 부문 Winner를 받았다.

기억이 더 흐려지기 전에, 그때 고민했던 기술적 도전과 레이아웃 구조를 기록해둔다.

## 프로젝트 개요

- **개발 기간:** 2011.02 ~ 2011.06
- **담당 역할:** 사용자 UI 개발, 인터랙션 모션 구현 (ActionScript 3.0)
- **목표:** 전국 쉐보레 영업소를 방문한 고객이 46인치 멀티 터치 키오스크로 e-카탈로그를 넘겨보고 디지털 오토 가이드를 확인하며 브랜드를 경험할 수 있는 오프라인 UI 애플리케이션

## 레이아웃과 아키텍처

웹 브라우저가 아니라 46인치 키오스크 하드웨어라는 특수한 환경이었다. 그래서 UI 전체를 독립형 컴포넌트 구조로 설계했다.

- **컴포넌트화.** 메인 대시보드, 차량 라인업 쇼룸, e-카탈로그 뷰어, 이벤트 인풋 폼 등 화면 단위를 각각 독립된 클래스(`Sprite`, `MovieClip`)로 분리했다.
- **이벤트 주도형 상태 관리.** 하위 컴포넌트끼리 직접 참조하지 않고, 커스텀 이벤트(`Event.BUBBLES`) 체인으로 네비게이션과 데이터를 제어해 결합도를 낮췄다.
- **해상도 대응.** 절대 좌표 대신 가변형 그리드 계산 인터페이스를 넣어서, 대형 디스플레이의 스케일링과 배치 유연성을 확보했다.

## 오프라인 키오스크 환경의 기술적 도전

매장에서 24시간 끊김 없이 돌아가야 하는 키오스크는 웹 개발보다 훨씬 엄격한 자원 관리가 필요했다.

### 메모리 누수 잡기

웹페이지는 새로고침하면 메모리가 초기화되지만, 키오스크는 며칠씩 켜져 있어야 해서 작은 누수도 치명적이었다.

- 모든 `addEventListener`에 `useWeakReference=true`를 강제해서, 가비지 컬렉터가 미사용 객체를 제때 수집하도록 했다.
- 화면 전환으로 제거되는 컴포넌트는 타이머(`Timer.stop()`), 비디오 스트림, 비트맵 배열을 명시적으로 해제하는 소멸자 메서드를 거치게 만들었다.

### 렌더링 퍼포먼스

46인치 화면에서 차량 이미지와 인터랙션이 60fps로 흐르도록 최적화했다.

- 고해상도 차량 벡터 그래픽과 정적 UI 레이어에는 `cacheAsBitmap = true`를 선언해 드로우 콜 부담을 줄였다.
- 알파 렌더링(`alpha < 1`) 겹침이 많은 터치 인터랙션 구간을 따로 골라내 프레임 드랍을 잡았다.

### 오프라인 데이터 싱크

영업소 네트워크가 불안정해도 안정적으로 동작하도록, 대용량 미디어와 XML 메타데이터를 로컬에 미리 불러오고 비동기 큐로 처리했다. 덕분에 터치했을 때 즉각적인 반응을 보장할 수 있었다.

## 마치며

ActionScript 3.0의 성능을 한계까지 시험해본 시간이었다. 생소한 대형 터치 UX 규격과 메모리 누수를 붙잡고 씨름한 끝에, 전국 쉐보레 매장에 키오스크가 놓인 걸 봤을 때는 기분이 남달랐다.

디자이너, 기획자들과 밤낮없이 만든 결과물이 2011 앤어워드 Winner로 이어지면서 그 해 상반기를 잘 마무리했다. 완성된 키오스크가 매장에 실제로 놓인 걸 봤을 때, 그 여러 밤을 새우며 붙잡고 있던 메모리 누수와 프레임 드랍이 감쪽같이 안 보인다는 사실이 오히려 가장 뿌듯했다.
