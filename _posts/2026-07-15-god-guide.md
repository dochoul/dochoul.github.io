이번에 회사에서 **GOD(G*** Open Design)**라는 사내 AI 디자인 툴을 만들었다. 이름 그대로 "설명만 하면 디자인이 되는" 도구인데, 밑바닥부터 새로 만든 건 아니고 오픈소스 프로젝트를 가져와 회사만의 도구로 바꾼 것이다.

### Open Design이란?

바탕이 된 건 [Open Design](https://github.com/nexu-io/open-design)이라는 오픈소스 프로젝트다. "Claude Design의 오픈소스 대안"을 표방하는 로컬 우선(local-first) vibe design workspace로, Apache-2.0 라이선스로 공개되어 있다. Claude Code, Codex, Cursor, Gemini CLI 등 21개 이상의 코딩 에이전트를 런타임으로 붙이는 agent-native 구조를 가지고 있고, 217개 이상의 플러그인과 151개 이상의 디자인 시스템이 기본으로 딸려 있다. 데스크톱 앱과 daemon이 전부 로컬에서 돌고, 생성 결과물도 프로젝트 디렉터리에 그대로 쌓인다.

방향성은 정확히 우리가 원하던 것이었다. 다만 이대로 회사에 풀기엔 두 가지가 걸렸다. 하나는 인증이었다. BYOK(Bring Your Own Key) 방식이라 직원마다 개인 API 키를 발급받아 써야 했다. 다른 하나는 배포였다. 만든 결과물을 어디에 올릴지가 빠져 있었다. 그래서 오픈소스를 그대로 쓰는 대신, 회사 환경에 맞게 고쳐 쓰기로 했다.

### 회사만의 것으로 바꾸며 더한 것들

이 프로젝트에서 내가 맡은 건 크게 네 가지다.

- **사내 AI Hub 연동**. 개인 API 키 대신 사내 AI Hub 게이트웨이로 인증하도록 붙였다. 시크릿 키 한 번만 입력하면 되고, 토큰 발급·관리도 사내 체계를 그대로 따른다.
- **OneStack 배포 연동**. 원본에는 없던 기능이다. 만든 결과물을 API 키 하나로 사내 OneStack에 바로 배포해서 `<앱슬러그>.onestack.run` 주소로 외부에 공유할 수 있게 했다.
- **플러그인·디자인 시스템 확장**. 스위스 그리드, 네오 브루탈리즘, 글래스모피즘, Dark Tech/Terminal 등 회사에서 자주 쓰는 스타일을 플러그인 갤러리에 추가했다.
- **설치부터 배포까지 전체 파이프라인 구축**. 런타임 연결, 로컬 CLI 감지, 배포까지 이어지는 흐름을 처음부터 끝까지 설계하고 붙였다.

말로는 네 줄이지만, 실제로는 "직원이 아무 설정 없이 로컬 CLI만 켜면 바로 쓸 수 있게" 만드는 데 대부분의 시간을 썼다. 아래는 지금 이 도구가 실제로 어떻게 동작하는지, 처음 실행부터 배포까지 직접 따라가 본 기록이다.

### 1. 온보딩과 런타임 연결

설치 후 처음 실행하면 '환영합니다' 온보딩 화면이 뜨고, 여기서 AI와 연결할 런타임을 고르면 바로 시작할 수 있다. 고를 수 있는 건 사내 AI Hub, 로컬 코딩 에이전트, 내 키 사용하기 세 가지인데, 가장 쉽고 빠른 방법은 로컬 코딩 에이전트 연결이다.

로컬 코딩 에이전트는 로컬에 설치된 CLI(Claude Code, Codex, Cursor, Gemini, OpenCode)를 통해 AI와 연결한다. 선택하면 PATH 환경 변수 스캔으로 설치된 CLI를 자동 감지(보통 5~10초)하며, 감지된 CLI를 골라 바로 사용할 수 있다.

- ✅ 별도 토큰·외부 키 불필요. 로컬에 설치된 CLI만 있으면 된다.
- ✅ 파일 읽기·쓰기·수정 가능. 프로젝트 파일을 직접 편집할 수 있다.

![런타임 선택 화면](/assets/god-guide/step2-runtime-select.png)

로컬에 Claude Code가 없다면 터미널에서 설치하면 된다.

```bash
# 사전 확인 — Node.js 18 이상 필요
node -v

# Claude Code 설치
sudo npm install -g @anthropic-ai/claude-code

# 설치 확인
which claude
```

`/usr/local/bin/claude` 경로가 나오면 성공이다. GOD 설정 화면으로 돌아가 다시 스캔을 누르면 Claude Code가 목록에 나타난다.

### 2. 사내 AI Hub로 연결하기

로컬 CLI 없이도 쓸 수 있도록 만든 게 사내 AI Hub 연동이다. 시크릿 키만 입력하면 사내 게이트웨이를 통해 Claude 등 모델을 바로 쓸 수 있다.

다만 AI Hub를 그냥 연결하면 채팅 모드로 동작한다. 텍스트·HTML 생성과 대화만 가능하고 프로젝트 파일을 직접 편집하지는 못한다는 뜻이다. 화면에도 이렇게 안내된다.

> 채팅 모드는 텍스트·HTML로만 답하고 프로젝트 파일을 편집할 수 없습니다. AI Hub로 파일을 읽고·쓰고·수정하려면 AI Hub에 연결된 로컬 Claude Code 에이전트로 전환하세요.

그래서 화면 하단의 '파일 편집 켜기 (Claude Code + AI Hub)' 토글을 켜 두는 걸 권장한다. 이 토글은 AI Hub로 로컬 Claude Code 에이전트를 실행해 프로젝트 파일을 편집하도록 전환해 준다.

![사내 AI Hub 연결 및 파일 편집 켜기 토글](/assets/god-guide/step3-aihub-toggle.png)

| 구분          | 파일 편집 끄기 (기본 채팅 모드)           | 파일 편집 켜기 (Claude Code + AI Hub)                     |
| ------------- | ----------------------------------------- | --------------------------------------------------------- |
| 동작 방식     | AI Hub 게이트웨이로 텍스트·HTML 생성/대화 | AI Hub가 로컬 Claude Code 에이전트를 실행해 파일까지 편집 |
| 할 수 있는 일 | 디자인·문서·프로토타입 생성 결과 확인     | 결과를 실제 파일에 반영, 코드/문서 직접 읽고·쓰고·수정    |
| 사전 준비     | AI Hub 토큰                               | AI Hub 토큰 + 로컬 Claude Code(CLI) 설치                  |

이 기능을 켜려면 로컬에 Claude Code(CLI)가 설치되어 있어야 한다. 설치되어 있지 않으면 토글을 켜도 파일 편집이 동작하지 않는다.

### 3. 프롬프트로 디자인 만들기

연결이 끝나면 메인 홈 화면으로 진입한다. 중앙 프롬프트 입력창에 원하는 디자인을 자연어로 설명하면 되고, 하단 플러그인 갤러리에서 원하는 스타일을 골라 바로 적용할 수 있다. 카드를 클릭하면 Details / Use 버튼이 나타나고, Use를 누르면 해당 플러그인이 프롬프트 입력창 상단에 태그로 붙는다.

아래 화면은 Dark Tech / Terminal 플러그인을 선택한 뒤 다음과 같이 작성한 예시다.

```
Dark Tech / Terminal 디자인 시스템(VoltAgent 계열 — 순수 블랙 캔버스 + 단일 에메랄드 그린 액센트 + 모노스페이스 정체성 + flat/border 위계)으로 랜딩 페이지를 만들어줘.

[제품]
사내 AI 에이전트 플랫폼 "회사 Agent". 직원이 GitLab/Hiworks/Mattermost를 연결해 업무 자동화 워크플로우를 직접 만드는 도구.

[타깃]
개발자 + 자동화에 관심 있는 비개발 직군. 둘 다 "터미널 켜고 바로 써보고 싶게" 만드는 톤.

[섹션 구성]
1. Hero — 한 줄 카피 + 터미널 프롬프트 형태의 실행 예시(코드 블록이 히어로)
2. 기능 3개 — RSS 뉴스봇, n8n 워크플로우, 사내 SSO 연동. 각각 짧은 코드/명령어 스니펫 포함
3. 아키텍처 다이어그램 — 에이전트가 도구를 호출하는 흐름 (border 기반, 그림자 없이)
4. 도입 가이드 — 설치/연결 3스텝, 명령어 위주
5. CTA — "지금 에이전트 켜기"

[톤 & 비주얼]
- 새벽 2시 IDE 같은 분위기. 친근한 SaaS 톤 금지, 엔지니어링 자신감
- 그린 액센트는 헤드라인/border/인터랙티브 요소에만 surgical하게
- glow는 액센트 요소에만, 본문은 따뜻한 그레이
- 코드 스니펫은 장식이 아니라 주인공
- 한국어 카피, 모노스페이스 강조

example.html로 만들고 데이터/카피만 바꿔서 재사용 가능하게.
```

작성이 끝나면 오른쪽 하단 전송 버튼(↑)을 눌러 생성을 시작한다.

### 4. 결과물과 자동 크리티크

전송 버튼을 누르면 GOD가 생각(Thinking) → 검색 → 호출 → 작성 단계를 거쳐 결과물을 생성한다. 예시에서는 약 8분 만에 37,131 토큰을 소비하며 두 개의 파일(`gabia-agent-landing.html`, `example.html`)이 만들어졌다.

순수 블랙 배경에 에메랄드 그린 액센트, 전면 모노스페이스 타이포그래피가 적용된 랜딩 페이지가 완성되었고, 히어로 영역 하단에는 실제 터미널 실행 예시가 코드 블록으로 배치되어 있다. 결과물과 함께 P0 체크 + 5-dim 크리티크도 자동으로 제공되는데, 디자인 원칙 준수 여부를 스스로 점검해 총점(예: 4.4/5)을 매기고 개선 포인트까지 짚어준다.

![생성된 랜딩 페이지 결과물과 P0 체크·5-dim 크리티크](/assets/god-guide/step5-result.png)

### 5. OneStack에 배포하기 (내가 붙인 부분)

원본 Open Design에는 없던 기능이다. 결과물이 마음에 들면 우측 상단 공유 버튼을 눌러 바로 외부에 배포할 수 있게 만들었다.

- **온라인에 게시**: OneStack에 배포 / Vercel에 다시 배포 / Cloudflare Pages에 배포
- **다운로드**: PDF·PPTX로 내보내기 / .zip 다운로드 / 독립 실행형 HTML로 내보내기 / Markdown으로 내보내기
- **저장**: 템플릿으로 저장

![공유 및 배포 옵션](/assets/god-guide/step6-share.png)

OneStack에 배포를 선택하면 API 키 입력 화면이 나타난다. OneStack API 키를 입력하고 저장을 누르면 `<앱슬러그>.onestack.run` 주소로 자동 배포된다. 배포가 완료되면 준비됨 상태와 함께 배포된 URL을 바로 확인하고 열 수 있다.

![OneStack 배포 완료 화면](/assets/god-guide/step6-deploy.png)

### 마치며

Open Design이라는 좋은 오픈소스가 이미 있었기 때문에, 처음부터 디자인 툴을 새로 만드는 대신 인증과 배포라는 우리 회사에 꼭 필요했던 두 조각만 채워 넣을 수 있었다. 덕분에 훨씬 짧은 시간 안에 실제로 쓸 수 있는 사내 도구가 나왔다.

디자인을 몰라도 괜찮다. 만들고 싶은 것을 말로 설명하면 GOD가 나머지를 처리한다. 피그마를 열지 않아도, 코드를 직접 짜지 않아도, 프롬프트 하나면 랜딩 페이지가 배포까지 완료된다. 오픈소스에서 시작했지만, 이제는 회사 사람들의 일하는 방식을 바꾸는 우리만의 도구가 됐다.
