남이 만든 코드도 아니고 내가 애지중지 키워온 프로젝트인데, 가끔은 객관적인 시선이 필요할 때가 있다.

Million.js 제작자가 만든 **`react-doctor`**를 알게 되었는데, 이걸 단순히 터미널에서 돌리는 게 아니라 **Claude Code의 Skill**로 설치해서 연동해 봤다. AI 에이전트가 내 코드의 안티 패턴을 직접 스캔하고 진단하게 만드는 과정은 꽤 흥미로운 경험이었다.

대상은 현재 작업 중인 프로젝트(`dashboard.office.hiworks.com`). 린트(Lint)와는 또 다른 느낌의 '건강검진' 결과는 생각보다 매웠다.

### 진단 결과: 62 / 100

![react-doctor](/assets/react-doctor/react-doctor.png)

나름 관리한다고 했는데 점수는 62점. "관리가 필요한 수준"이라는 진단을 받았다. 총 537개의 이슈가 쏟아졌는데, 단순히 "이거 틀렸어"라고 지적질하는 게 아니라 Critical / Warning / Dead Code로 나누어 우선순위를 짚어주는 게 꽤 친절하다.

이번 검진에서 발견된 주요 '환부'들은 이렇다.

#### 1. 중첩 컴포넌트

`FoldContent.tsx` 내부에서 다른 컴포넌트를 정의해서 쓰고 있었다. 리렌더링될 때마다 자식 인스턴스가 새로 생성되니 성능도 떨어지고 state도 자꾸 초기화되던 범인이었다. "이럴 시간에 그냥 분리하자"는 말이 절로 나왔다.

#### 2. 새어 나가는 메모리

`CalendarWidget.tsx`에서 `setTimeout`을 돌리면서 정작 `clearTimeout`은 안 하고 있었다. 전형적인 메모리 누수 패턴인데, Claude가 스킬을 통해 이걸 귀신같이 잡아냈다.

#### 3. 불필요한 코드들

미사용 export가 164개, 미사용 파일이 72개나 나왔다. 기능 추가할 땐 신나게 만들고, 안 쓸 땐 그냥 방치해둔 흔적들이다. 이것만 걷어내도 프로젝트가 한결 가벼워질 것 같다.

### 직접 해보려면? (사용법)

궁금한 분들을 위해 간단한 사용법을 정리했다. 별도의 복잡한 설치 과정도 필요 없다.

#### 1. 가장 빠른 방법 (npx)

별도 설치 없이 프로젝트 루트에서 아래 명령어를 실행하면 된다.
```bash
npx react-doctor@latest .
```

#### 2. Claude Code 스킬로 활용하기

나처럼 Claude Code를 사용 중이라면, 아예 스킬(Skill)로 등록해두고 사용하는 게 훨씬 편하다. 에이전트가 진단 결과를 직접 읽고 리팩토링까지 제안할 수 있기 때문이다.

먼저 아래 명령어로 스킬을 설치한다.
```bash
npx -y react-doctor@latest install
```

설치 후 Claude Code에서 "react doctor로 검수해줘"라고 말하면 AI가 알아서 스캔을 시작한다.

### 총평: AI 에이전트와 함께하는 정기검진

react-doctor를 Claude Code의 스킬로 사용했을 때의 가장 큰 장점은 **'진단 후 즉각적인 처방'**이 가능하다는 것이다. 에이전트가 리포트를 읽고 어디부터 손대야 할지 명확하게 인지하고 있으니, 내가 일일이 코드를 찾을 필요 없이 즉시 리팩토링으로 이어갈 수 있었다.

내 코드가 완벽하다고 믿고 싶지만, 가끔은 이렇게 외부의 시선으로 뼈를 맞을 필요가 있다. 62점이라는 점수가 좀 쓰리긴 했지만, Claude에게 수정해달라고 했더니 하나씩 바로바로 고쳐줬다. 진단부터 처방까지 한 흐름으로 이어지니 생각보다 훨씬 빠르게 정리됐다.

React 프로젝트를 다루는 프론트엔드 개발자라면 한 번쯤 돌려볼 만하다.  
내 코드가 얼마나 건강한지, 외면해온 것들이 얼마나 쌓였는지 숫자로 보고 나면 생각이 달라진다.
